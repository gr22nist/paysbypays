# 데이터 폴더 구조

이 폴더는 대시보드에서 사용하는 시드 데이터와 헬퍼 함수를 관리합니다.

## 구조

### `health.ts`
시스템 헬스 모니터링용 시드 데이터를 제공합니다.
- `mockHealthResponse`: 오프라인 개발 시나리오를 위한 헬스 응답 데이터
- `buildMockHealthPayload()`: 동적으로 헬스 페이로드를 생성하는 함수

### `settlements.ts`
정산 관련 시드 데이터와 타입 정의를 제공합니다.
- `SettlementRecord`: 정산 내역 레코드 타입
- `SettlementSummary`: 정산 요약 통계 타입
- `mockSettlementRecords`: 정산 시드 데이터 배열
- `METHOD_FEE_MAP`: 결제 수단별 수수료 비율 매핑

### `pay-types.ts`
결제 수단 메타데이터를 제공합니다.
- `getPayTypeMeta()`: 결제 수단 코드에 대한 메타데이터 반환

### `merchant-status.ts`
가맹점 상태 관련 상수와 타입을 제공합니다.

## 사용 원칙

1. **실제 API 우선**: 실제 API가 제공되는 경우, API 데이터를 우선 사용합니다.
2. **시드 데이터 보완**: API 데이터가 부족하거나 오프라인 환경에서는 시드 데이터로 보완합니다.
3. **타입 안정성**: 모든 데이터는 TypeScript 타입으로 정의되어 있습니다.
4. **일관성**: 실제 API 응답 구조와 시드 데이터 구조를 일치시킵니다.

## 확장 가이드

새로운 시드 데이터를 추가할 때:

1. 타입 정의를 먼저 작성합니다.
2. 시드 데이터 배열/객체를 생성합니다.
3. 필요시 헬퍼 함수를 추가합니다.
4. 이 README에 문서화합니다.
